Research

I've been exploring cloud security, data autonomy, and privacy concerns. This research led me to propose a new model for local, private cloud systems — combining the convenience of cloud with the control of on-premise infrastructure.

Mixed-Methods Study (2024–2025)

I conducted research analyzing breach trends and trust gaps in cloud platforms. This work led me to propose the "Local Centralized Private Cloud Model" — a new approach to data security and autonomy.

Quantitative Analysis

I gathered breach data from 200 companies across industries (2020–2023), then organized and analyzed it in Excel using pivot tables and charts.

  • Common breach vectors: credential theft, misconfigurations, insider access
  • Authentication failures were connected to major losses
  • Cloud misconfigurations kept appearing as a top issue

Qualitative Analysis

I ran a survey on LinkedIn with IT professionals using a Likert scale, asking about their trust in and concerns about public cloud providers.

  • Participants wanted more transparency in how cloud providers handle data
  • Many worried about vendor lock-in and unclear security practices
  • Significant interest in local, private cloud alternatives

The Local Centralized Private Cloud Model

A new architecture designed to merge cloud-style convenience with local-hardware autonomy.

Concept

A self-contained system running locally on user-owned hardware, providing cloud capabilities without dependency on public cloud vendors.

Key Benefits

  • Data autonomy and physical control
  • Reduced attack surface
  • No vendor lock-in
  • Predictable cost structure

Security Stack

  • RAID-1 redundancy
  • Encryption at rest + in transit
  • Strict local-only access model
  • Firewall + segmentation controls

Future Vision

Integrating workflow engines (n8n), NAS features, container services, and local AI models into a single trustworthy edge appliance.
